# 카페 댓글 자동화 운영 가이드

> 이 문서는 리포에 저장돼 있어 **다른 PC에서 `git pull` 하면 그대로 봅니다.**
> 댓글 작성은 사장님 Windows PC(한국 IP)의 **로그인된 크롬**에서 돌아야 합니다.
> 크롤링 운영은 [크롤링-운영.md](크롤링-운영.md) 참고 (별개 시스템).

## 1. 구성 (3종이 동시에 떠 있어야 함)

| 프로세스 | 실행 파일 | 역할 |
|---|---|---|
| 헤드리스 크롬 | `crawler/cafe_cmt/run_chrome.bat` (포트 **9224**) | 로그인 세션 유지 (발행용 9223과 분리) |
| 댓글 게시 데몬 | `crawler/cafe_cmt/comment_listener.py` | `cafe_comment_queue` 폴링 → 실제 댓글 게시 |
| 새 글 감시 워처 | `crawler/cafe_cmt/watch_new_posts.py` | 등록 카페 5분마다 크롤 → 새 글에 댓글 예약 |

- **원클릭 시작**: `crawler/cafe_cmt/start_all.bat` (3종 한번에)
- **종료**: `crawler/cafe_cmt/stop_all.bat`
- **부팅 자동시작**: 시작프로그램 폴더에 `start_all.bat` 바로가기 등록됨
  (`%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup\DDMKT-카페댓글자동화.lnk`)

## 2. ⚠️ PC 절전/잠금 — 여기가 제일 많이 터지는 곳

**조건: PC 켜둠 + 로그인 유지.** 크롤링과 동일하게 **모니터 꺼짐·화면 잠금은 괜찮습니다**
(헤드리스 크롬은 데스크톱 표시가 필요 없고, `_focus_naver_window()` 실패는 try/except로 무해).

**하지만 절전(sleep)/최대 절전은 치명적** — 들어가는 순간 크롬·데몬·워처가 전부 멈춥니다.

### 적용해 둔 전원 설정 (2026-07-18, 데스크톱/AC)
```powershell
powercfg /change standby-timeout-ac 0     # 절전 사용 안 함
powercfg /change hibernate-timeout-ac 0   # 최대 절전 사용 안 함
powercfg /change disk-timeout-ac 0        # 하드디스크 끄기 안 함
```
확인:
```powershell
powercfg /query SCHEME_CURRENT SUB_SLEEP STANDBYIDLE    # AC 색인 0x00000000 이어야 함
powercfg /query SCHEME_CURRENT SUB_SLEEP HIBERNATEIDLE
```
> 모니터 끄기(기본 30분)는 그대로 둬도 됩니다. 화면 잠금도 무관.

### 그 외 프로세스가 죽는 원인
- **터미널/세션 종료**: 데몬을 어떤 세션에 물려서 띄우면 그 세션이 끝날 때 같이 죽습니다.
  → 반드시 `start_all.bat`(독립 콘솔) 또는 부팅 자동시작으로 띄울 것.
- **네이버 로그인 만료**: 워처가 5분마다 카페를 열어 세션을 데워 주지만, 만료되면
  `run_chrome_login.bat`로 1회 재로그인 필요. (데몬은 `LOGIN_REQUIRED`를 감지해 작업을 `pending`으로 되돌려 두므로 재로그인하면 자동 재개)

## 2-1. ⚠️ `.bat` 작성 규칙 (2026-07-18 실제 디버깅으로 확정)

`start_all.bat`이 **조용히 아무것도 안 뜨는** 사고가 있었고, 원인이 4개 복합이었습니다.
새 bat을 만들거나 고칠 때 아래를 반드시 지킬 것:

| 규칙 | 안 지키면 |
|---|---|
| **ASCII만** (한글 주석·창제목 금지) | cmd/스케줄러 코드페이지 깨져 exit 255로 즉사 |
| **CRLF 줄바꿈** (`.gitattributes`가 강제하나 새로 만들 때 확인) | cmd가 파싱 못 해 자동시작이 조용히 실패 |
| **`%~dp0`의 끝 백슬래시 제거 후 사용** | `"C:\dir\"` 안의 `\"` 때문에 `cd`·`start /d`가 조용히 실패 |
| **`timeout` 대신 `ping -n N 127.0.0.1 >nul`** | stdin 리디렉션 시 `timeout`이 즉시 에러로 중단 |

끝 백슬래시 제거 관용구:
```bat
set "HERE=%~dp0"
if "%HERE:~-1%"=="\" set "HERE=%HERE:~0,-1%"
cd /d "%HERE%"
```
python은 `cmd /k` 중첩 따옴표로 감싸지 말고 **직접 실행**할 것: `start "title" /min py script.py`

## 3. 상태 확인

```powershell
# 3종 생존 확인
Get-CimInstance Win32_Process -Filter "Name='python.exe'" |
  Where-Object { $_.CommandLine -match 'comment_listener|watch_new_posts' } |
  Select-Object ProcessId, CommandLine
Test-NetConnection 127.0.0.1 -Port 9224    # 크롬 CDP
```
- 웹: 사이드바 **카페 › 카페 댓글 자동화** 페이지에서 예약/상태 확인
- 감시 카페 목록: `cafe_comment_watch` 테이블 / 댓글 큐: `cafe_comment_queue`

## 4. 안전장치 (계정 보호)
- **한 글에 댓글 1번만** — 글번호 기준 중복 차단(`already_commented`)
- **댓글 간 최소 간격** — `CAFE_CMT_MIN_GAP_MIN`(기본 20분)
- **첫 크롤은 기준선만** — 등록 직후 기존 글에 몰아서 달지 않음
- **한 번에 예약 상한** — `CAFE_CMT_WATCH_MAX`(기본 5건)

## 자리 비울 때 (무인 운영) — 2026-07-20 추가

### 1. 다른 PC 크롤과 겹치지 않게 — 정지 시간대
회사 PC의 **전체크롤이 평일 03:00~09:00**(`docs/크롤링-운영.md`)에 돕니다.
리스너는 이 시간대엔 **게시를 멈춥니다**(큐는 그대로 쌓이고, 끝나면 이어서 처리).

```
CAFE_CMT_QUIET=03:00-09:00     (기본값. 끄려면 CAFE_CMT_QUIET= 로 비우기)
```
자정을 넘는 구간(`23:00-06:00`)도 지원합니다.

> 실제로는 PC·IP·대상 사이트가 모두 달라 크롤과 물리적 충돌은 없습니다.
> 그보다 **새벽 4시에 댓글이 달리는 게 부자연스러워서** 두는 장치에 가깝습니다.

### 2. 프로세스가 죽어도 살아나게 — 감시기
`watchdog.ps1` + 예약 작업 **DDMKT-CafeComment-Watchdog**(5분 간격).

| 감시 대상 | 죽었을 때 |
|---|---|
| `comment_listener.py` / `watch_new_posts.py` / `reply_scheduler.py` | 다시 띄움 |
| 계정별 크롬(9224~9227) | 프로필 잠금 정리 후 `run_chrome.bat <계정>` |

- 로그: `crawler/cafe_cmt/watchdog.log` (되살린 기록만 남음 — 평소엔 비어 있음)
- ⚠️ **발행 리스너(9223)는 건드리지 않습니다.** 담당이 달라 중복 발행 위험이 있습니다.
- 등록 확인: `Get-ScheduledTaskInfo -TaskName 'DDMKT-CafeComment-Watchdog'` → `LastTaskResult 0`

재등록이 필요하면:
```powershell
$H='C:\Users\rlawh\DDMKT_ERP\crawler\cafe_cmt'
$a=New-ScheduledTaskAction -Execute 'powershell.exe' -Argument "-ExecutionPolicy Bypass -NoProfile -WindowStyle Hidden -File `"$H\watchdog.ps1`"" -WorkingDirectory $H
$t=New-ScheduledTaskTrigger -Once -At (Get-Date).AddMinutes(1) -RepetitionInterval (New-TimeSpan -Minutes 5)
$s=New-ScheduledTaskSettingsSet -StartWhenAvailable -MultipleInstances IgnoreNew -ExecutionTimeLimit (New-TimeSpan -Minutes 10)
$p=New-ScheduledTaskPrincipal -UserId $env:USERNAME -LogonType Interactive -RunLevel Limited
Register-ScheduledTask -TaskName 'DDMKT-CafeComment-Watchdog' -Action $a -Trigger $t -Settings $s -Principal $p -Force
```

### 3. 절전으로 멈추지 않게
```
powercfg /change standby-timeout-ac 0
powercfg /change hibernate-timeout-ac 0
powercfg /change disk-timeout-ac 0
```
확인: `powercfg /q SCHEME_CURRENT SUB_SLEEP STANDBYIDLE` → `0x00000000`
(모니터 끄기·화면 잠금은 영향 없음. **로그오프·종료는 끊김**.)

### 4. 과거 글 댓글 일괄 보충
`reply_scheduler` 의 조회 범위가 60건이면 밀린 글의 오래된 댓글이 창 밖으로 밀려
**영영 답글을 못 받습니다.** 기본값을 300으로 올려뒀습니다(`CAFE_CMT_REPLY_LOOKBACK`).

일괄 보충 시 주의: 한꺼번에 넣으면 같은 시각에 몰려 티가 납니다.
글마다 25분, 계정마다 8분 + 지터로 분산하고, 정지 시간대는 **건너뛰되 간격은 유지**해야
합니다(그냥 "정지면 09:05로 당김" 으로 하면 그 구간 물량이 09:05에 전부 뭉칩니다).
