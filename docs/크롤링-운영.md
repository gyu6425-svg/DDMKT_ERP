# 크롤링 운영 가이드 (집/회사 공용)

> 이 문서는 리포에 저장돼 있어 **집에서 `git pull` 하면 그대로 봅니다.**
> 크롤은 사장님 Windows PC(한국 IP)에서 돌아야 네이버 지오블로킹·통합탭 차이가 없습니다.

## 1. 자동 스케줄 (회사 PC · Windows 작업 스케줄러)

| 작업 이름 | 언제 | 실행 파일 | 내용 |
|---|---|---|---|
| **DDMKT-Crawl-Full** | 평일(월~금) **03:00** | `crawler/run_crawler.bat` | 전체 추적글 크롤 (`--spread --deadline 09:00`, 6시간 여유) |
| **DDMKT-Crawl-Today-WD** | 평일 09:05부터 **30분 간격**(~24시) | `crawler/run_today.bat` | 당일 발행글 반복 크롤 (`crawl_bydate.py 1`) |
| **DDMKT-Crawl-Today-WE** | 주말(토·일) **90분 간격** | `crawler/run_today.bat` | 주말 당일글 (전체크롤은 안 함) |
| **DDMKT-Crawl-Place** | **매일 09:20** | `crawler/run_place.bat` | 플레이스 순위 측정 (`place_rank_crawler.py`) |

- **월요일 새벽 3시 전체크롤** = DDMKT-Crawl-Full (평일에 포함). 일요일 밤~월요일 03:00에 도는 건 아니고, **월요일 03:00에 시작**해 09:00 전 완료.
- 조건: **PC 켜둠 + 로그인 유지**(작업이 `/IT` 대화형이라 로그인 세션 필요). 모니터·잠금은 괜찮음.

### ⚠️ 절대 규칙
1. **`.bat`은 ASCII(영문)만** — 한글 주석 있으면 작업 스케줄러 코드페이지에서 깨져 **exit 255로 즉사**.
2. **각 `.bat`은 python을 `start /min` 으로 "자기만의 새 콘솔"에서 띄운다**(2026-07-13 적용). 이렇게 안 하면, 뒤에 뜬 작업(당일글 09:05·플레이스 09:20)이 같은 세션 콘솔을 공유하는 전체크롤(03시)에 **Ctrl+C를 보내 `0xC000013A(-1073741510)`로 죽인다**(실제 2026-07-13 03시 크롤이 72/117에서 이렇게 죽음). 격리 후 서로 안 죽음.
   - 트레이드오프: `start`는 즉시 반환이라 작업이 바로 "완료"로 뜨고 python은 새 콘솔에서 계속 돎(진행률은 앱 크롤현황/`crawl_status`, 로그는 `crawler.log`로 확인).
3. 작업 설정 변경은 incremental 말고 **Register-ScheduledTask -Xml -Force로 통째 재등록**.

## 2. 집에서 수동으로 돌리기 (주말·긴급용)

집 PC(한국 IP)에서도 아래처럼 돌릴 수 있습니다. **먼저 준비물**:

- Python 3.14 + 패키지: `pip install requests truststore`
- `crawler/.env` 파일에 아래 두 값 설정 (⚠️ 키는 회사 PC의 `crawler/.env`에서 복사 — **깃에 올리지 말 것**, gitignore됨):
  ```
  SUPABASE_URL=...
  SUPABASE_SERVICE_KEY=...
  ```

### 명령어 (crawler 폴더에서)
```bash
cd crawler

# ① 플레이스 순위 체크 (주말에 집에서 돌릴 것 — 등록 업체 전부, 하루 1회)
python place_rank_crawler.py

# ② 당일 발행글 크롤 (오늘분)
python crawl_bydate.py 1

# ③ 전체 추적글 크롤 (시간 분산)
python blog_rank_crawler.py --spread --chunk-size 5 --deadline 09:00

# ④ 특정 업체만 집중 재크롤 (오늘+어제) — 예: 양산 샘이깊은국어학원
python crawl_pick.py "양산 샘이깊은국어학원"
```

- 플레이스는 **하루 1회 측정**(이미 측정한 날은 자동 스킵). 결과: `place_keywords.measurements`에 `{date, rank, status}` 저장.
- 당일글/전체 크롤은 `crawl_status`(대시보드 크롤 현황)에 진행률·최근 실행이 기록됩니다.

## 3. 상태 확인
- 앱 대시보드의 **크롤 현황**(당일 측정 글 등)에서 확인.
- 로그: `crawler/crawler.log` (각 실행의 START/END + exit 코드).
- 스케줄 작업 점검(회사 PC):
  ```bash
  schtasks /query /tn "DDMKT-Crawl-Full" /fo LIST /v
  schtasks /query /tn "DDMKT-Crawl-Place" /fo LIST /v
  ```
