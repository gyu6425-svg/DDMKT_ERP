# PC 상시 가동 설정 (자동화가 24시간 안 끊기게)

크롤링·카페 발행·댓글 자동화는 **PC가 켜져 있고 프로그램이 살아 있어야** 돌아갑니다.
이 문서는 **메인 PC에 실제로 적용된 설정**을 기준으로, 다른 PC에도 똑같이 맞추는 방법입니다.

> 📌 **모니터만 끄는 건 괜찮습니다.** 모니터는 화면 출력일 뿐이라 PC·프로그램은 계속 돕니다.
> 위험한 건 **절전(잠자기)** 입니다. 절전에 들어가면 크롬과 파이썬이 멈춰서 자동화가 끊깁니다.

---

## 1. 전원 설정 — 절대 잠들지 않게

### 메인 PC 기준값 (이대로 맞추면 됨)

| 항목 | 전원 연결 | 배터리 |
|---|---|---|
| **절전(잠자기)** | **안 함** ⭐ | 안 함 |
| **최대 절전(하이버네이트)** | **안 함** ⭐ | 안 함 |
| 디스크 끄기 | 안 함 | 10분 |
| 모니터 끄기 | 안 함 | 3분 |

⭐ 표시된 **절전·최대절전 2개가 핵심**입니다. 나머지는 자동화에 영향 없습니다.
(모니터 끄기는 켜두셔도 됩니다 — 화면만 꺼지고 프로그램은 계속 돕니다.)

### 지금 내 PC 설정 확인하기

명령 프롬프트(cmd)에서:
```
powercfg /q SCHEME_CURRENT SUB_SLEEP STANDBYIDLE
```
`Current AC Power Setting Index: 0x00000000` 처럼 **0x00000000 이면 "안 함"** 입니다.
0이 아니면 아래로 바꾸세요.

### 설정 바꾸기 (권장: 클릭으로)

1. `윈도우키 + R` → `powercfg.cpl` 입력 → 엔터
2. 현재 선택된 전원 관리 옵션 옆 **"설정 변경"** 클릭
3. **"컴퓨터를 절전 모드로 전환"** → **"해당 없음"** 으로 변경
4. (노트북이면 "배터리 사용" 쪽도 같이 "해당 없음")
5. **변경 내용 저장**

### 설정 바꾸기 (명령어로 한 번에)

cmd를 **관리자 권한**으로 열고:
```
powercfg /change standby-timeout-ac 0
powercfg /change standby-timeout-dc 0
powercfg /change hibernate-timeout-ac 0
powercfg /change hibernate-timeout-dc 0
```
(`0` = 안 함)

## 2. Windows Update 자동 재시작 막기

**업데이트 후 자동 재시작이 자동화를 끊는 흔한 원인입니다.**
재시작 자체는 막을 수 없지만, **"활성 시간"** 을 지정하면 그 시간엔 재시작하지 않습니다.

### 메인 PC 기준값

```
활성 시간: 09시 ~ 22시
```

> ⚠️ 이 설정의 한계: 메인 PC는 **새벽 3시~8시 30분에 전체크롤**을 도는데,
> 이 시간대는 활성 시간(9~22시) **밖**이라 재시작이 일어날 수 있습니다.
> 크롤 PC라면 활성 시간을 넓게(예: 0~23시) 잡는 편이 안전합니다.

### 설정 방법
`설정` → `Windows 업데이트` → `고급 옵션` → **`활성 시간`** → 사용 시간대 지정

---

## 3. 화면 잠금은 괜찮지만, 로그오프는 안 됨

| | 자동화 영향 |
|---|---|
| 모니터 끄기 | ✅ 영향 없음 |
| 화면 잠금(Win+L) | ✅ 영향 없음 — 프로그램은 계속 돎 |
| **로그오프 / 사용자 전환** | ❌ **끊김** — 그 계정으로 뜬 프로그램이 다 종료됨 |
| **절전 / 최대절전** | ❌ **끊김** |
| **종료 / 재시작** | ❌ 끊김 (재시작 후 자동시작으로 복구는 됨 — 4번 참고) |

**퇴근할 때는 "모니터만 끄기" 또는 "화면 잠금"** 을 쓰세요. 절대 종료·로그오프하지 마세요.

## 4. 재부팅돼도 자동 복구되게 (시작프로그램)

PC가 꺼졌다 켜져도 자동화가 스스로 살아나게 하는 장치입니다.

### 시작프로그램 폴더 열기
`윈도우키 + R` → `shell:startup` → 엔터

### 여기에 넣는 파일 (역할별)

| 파일 | 역할 | 어느 PC에 |
|---|---|---|
| `DDMKT-Listener.vbs` | 순위 **재검색** 리스너 | 크롤 담당 PC |
| `DDMKT-Cafe.vbs` | 카페 **발행** 리스너 | 카페 담당 PC |

레포의 아래 위치에서 **복사**해 넣으면 됩니다:
```
DDMKT_ERP\crawler\cafe_pub\DDMKT-Cafe.vbs
```

> ⚠️ **역할이 겹치면 안 됩니다.** 카페 발행 VBS를 두 PC 시작프로그램에 다 넣으면
> **같은 글이 두 번 올라갑니다.** 담당 PC 한 곳에만 넣으세요.

### 자동 재시작 루프도 들어있음
`run_cafe_listener.bat`, `run_listener.bat` 안에는 `:loop` 가 있어서,
**파이썬이 어떤 이유로 죽어도 30초 뒤 자동으로 다시 뜹니다.**

> ⚠️ 그래서 **리스너를 끄려면 파이썬만 죽이면 안 됩니다.**
> bat을 돌리는 `cmd.exe` 까지 같이 종료해야 합니다. (안 그러면 30초 뒤 되살아남)
> 작업 관리자에서 `cmd.exe` 중 해당 bat을 실행 중인 것도 같이 종료하세요.

## 5. ⚠️ 자동시작이 조용히 죽는 함정 — `.bat`/`.vbs` 줄바꿈

**실제로 겪은 사고입니다.** 재부팅 후 크롤 리스너·카페 리스너가 **둘 다 안 돌았는데**,
시작프로그램 등록도 멀쩡하고 파일도 그대로 있어서 원인 찾기가 어려웠습니다.

**원인**: `.bat` / `.vbs` 파일이 **LF 줄바꿈(리눅스 방식)** 으로 저장돼 있으면
Windows `cmd`/`wscript` 가 해석을 못 해서 **아무 오류 없이 조용히 실패**합니다.

- **증상**: 로그 파일이 아예 안 생김 / `'Startup'은(는) 내부 또는 외부 명령이 아닙니다` 같은 엉뚱한 오류
- **확인**: 메모장 말고 VS Code로 열면 우측 하단에 `CRLF` 인지 `LF` 인지 표시됨
- **해결**: `CRLF` 로 바꿔 저장
- **예방**: 레포 루트 `.gitattributes` 에 이미 `*.bat *.cmd *.vbs *.ps1 → eol=crlf` 를 넣어뒀습니다.
  **git으로 받으면 자동으로 CRLF** 가 되므로 보통은 신경 안 쓰셔도 됩니다.

## 6. 카페 발행 담당 PC 추가 설정 — 네이버 로그인 유지

**카페 발행을 맡은 PC만 해당됩니다.**

### ⭐ "로그인 상태 유지" 반드시 체크

네이버 로그인 시 이걸 **체크하지 않으면** 로그인 정보가 크롬 메모리에만 남습니다.
→ **크롬이 꺼지거나 재부팅하면 로그인이 통째로 사라지고 발행이 멈춥니다.**

체크하면 디스크에 저장돼서 **크롬이 죽어도, 재부팅해도 로그인이 유지**됩니다.

> 실제로 이것 때문에 발행이 멈춘 적이 있습니다. 크롬 프로세스가 종료되는 순간
> 세션이 날아가서, 다음 발행 시각에 `LOGIN_REQUIRED` 로 실패했습니다.

### 로그인 살아있는지 확인
```
cd DDMKT_ERP\crawler\cafe_pub
python -c "import publish_cafe as pc; print(pc.session_ping(pc.DEFAULT_CDP))"
```
- `True` → 정상, 발행 가능
- `False` → 로그아웃됨 → `run_chrome_login.bat` 으로 재로그인
- `None` → 크롬이 안 떠 있음 → `run_chrome.bat` 실행

### 자동 세션 유지
발행 리스너는 **유휴 시 9분마다** 네이버에 접속해 세션을 살려둡니다(`CAFE_KEEPALIVE_MIN`).
만료를 감지하면 경고를 남기지만, **자동 재로그인은 하지 않습니다**
(캡차·2단계인증·계정 잠금 위험 때문에 일부러 그렇게 만들었습니다). 사람이 다시 로그인해야 합니다.

## 7. 설치 후 점검 체크리스트

새 PC를 상시 가동으로 만든 뒤 아래를 확인하세요.

- [ ] `powercfg /q SCHEME_CURRENT SUB_SLEEP STANDBYIDLE` → **0x00000000** (절전 안 함)
- [ ] Windows Update 활성 시간이 자동화 시간대를 덮고 있는지
- [ ] `shell:startup` 에 **담당 역할의 VBS만** 들어있는지 (겹치면 중복 실행)
- [ ] 재부팅 테스트 → 로그인만 하고 기다렸을 때 리스너가 자동으로 뜨는지
      (`crawler\cafe_pub\cafe_listener.log` 또는 `crawler\listener.log` 에 새 기록이 쌓이면 성공)
- [ ] (발행 PC) `session_ping` 이 `True` 인지
- [ ] 퇴근 시 **종료·로그오프하지 않기** — 모니터만 끄거나 화면 잠금

## 8. 프로그램이 살아있는지 한 번에 보기

명령 프롬프트에서:
```
tasklist | findstr python
```
- **크롤 담당 PC**: `run_listener.py` (재검색) 가 보여야 정상
- **카페 담당 PC**: `publish_listener.py` (발행), 댓글 자동화 프로세스가 보여야 정상

크롬 포트 확인:
```
netstat -ano | findstr "9223 9224"
```
- `9222` 카카오 / `9223` 카페 발행 / `9224` 이상 카페 댓글(계정별)
- `LISTENING` 이 보이면 해당 크롬이 떠 있는 것입니다.
