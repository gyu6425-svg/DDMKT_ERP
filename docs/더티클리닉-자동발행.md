## 1. 배너 지역 AI 생성 — ⚠️ 비용 $0.02 고정 (핵심)

기존 더맨/누수와 **동일한 방식**: OpenAI **Responses API + `image_generation` tool**.

- 기존 배너(`main_banner01.png`)를 `input_image`로 참조 + **"지역명만 교체"** 프롬프트
- **한글 정확 렌더**(코드 합성 아님, AI가 원본 스타일 유지하며 텍스트만 교체)
- **모델 `gpt-5-mini` · quality `low` · 1024 고정** → 항상 **$0.0136 < $0.02**
- ⚠️ **`gpt-5.5`를 쓰면 출력 토큰 단가가 $2→$42(21배)** 라 배너만 $0.05 초과.
  → 코드에서 **하드코딩으로 gpt-5-mini/low 고정**(`.env`가 gpt-5.5여도 무시). 절대 env 값 그대로 쓰지 말 것.

비용 근거(`src/lib/apiPricing.ts`): square/low = 272토큰 × $40/M = $0.0109 + 텍스트 토큰 ≈ $0.0027.

발행글 이미지 순서: `[지역배너, banner03, banner02, 1,4,5, banner06, 7,3, banner05, 6,2, banner04, banner07, banner08]`

## 3. 누락 방지 (⭐ 어제 '프로세스 죽음' 재발 방지)

| 방어 | 내용 |
|---|---|
| 각 건 독립 | 지역별 `try/except` — 한 건 실패해도 다음 계속 |
| 배너 재시도 | 배너 생성 3회 재시도(429·네트워크 일시오류) |
| **완료 기록** | 성공 지역을 `published_regions.json`에 기록 → **재시작 시 건너뛰고 이어감(중복·누락 방지)** |
| 실패 기록 | `failed_regions.json` → 나중 재시도 대상 |
| **watchdog** | `.bat`이 프로세스가 죽으면 재시작 → 완료분 스킵하고 남은 지역부터. 어제 리스너 크래시 같은 정지를 자동 복구 |

**watchdog `.bat` 예시** (`run_dirty.bat`):
```bat
@echo off
:loop
py -u auto_publish_dirty.py
REM 정상 종료(STOP_AT·상한 도달)면 errorlevel 0 → 멈춤. 비정상(크래시)이면 재시작.
if %errorlevel%==0 goto end
timeout /t 15 >nul
goto loop
:end
```
> 정상 종료(STOP_AT/상한)는 `exit 0`, 크래시는 비정상 종료 → .bat이 구분해 크래시만 재시작.

## 4. 실행 순서

1. **dev 서버**: `npm run api:dev` (배너 생성 API `:8787`)
2. **크롬 로그인**: `crawler/cafe_pub/run_chrome_login.bat` → 더티클리닉 발행 계정(네이버, "로그인 상태 유지" 체크)
3. **인기글 스캔**: `node scan_popular.mjs` → `passed_regions.json`
4. **발행**: `run_dirty.bat` (watchdog). 환경변수:
   - `DIRTY_MAX=6` (상한 건수) · `DIRTY_STOP_AT=23:59` · `DIRTY_GAP_MIN=30`/`DIRTY_GAP_MAX=42`(간격 분)

## 5. sub2 이식 절차

1. `git fetch && git checkout -B sub2 origin/main` — 스크립트·문서·고정 이미지 전부 받음
2. `.env`(로컬, gitignore): `OPENAI_API_KEY`, `OPENAI_IMAGE_MODEL`, `SUPABASE_*`,
   `crawler/cafe_pub/.env`의 `CAFE_WRITE_URL=<더티클리닉 카페 글쓰기 URL>` · `CAFE_BOARD=더티클리닉 입주청소`
3. `npm install` · `pip install playwright requests pillow truststore` · `python -m playwright install chromium`
4. §4 순서대로 실행
5. ⚠️ **리스너/발행은 한 대에서만** — 이 PC와 동시에 켜면 중복 발행([[multi-computer-git-workflow]] 규칙)
